KEY LEARNINGS & PRINCIPLES
================================================================================

1. Vision-reading only: Legacy font encoding makes raw text extraction unusable.
   Always read gita_pages/ PNG images for source content.

2. Regex is dangerous for JS edits: Use find_value_span() (char-by-char walk
   handling \\ escapes) for replacing verse values. re.sub() silently truncates.

3. TWO KN blocks in viewer-bundled.html: ~line 600 and ~line 6450. Any fix to
   bannanje_kn.js must also be applied to BOTH blocks in viewer-bundled.html.

4. Sync after every session: Standalone .js files must be synced into their
   inline blocks inside viewer-bundled.html after every authoring session.

5. Work in repo directly: Modify files in /home/claude/Bhagavadgita/, commit
   with git commit, and push. Never create separate output files.

6. Autonomous execution expected: Vinayak expects chapter-by-chapter completion
   without mid-task status checks. Plan → execute → log → report when done.

7. Structural gaps are real: 3.43 and 13.35 are genuine; 11.52–11.55 is not.

8. All costs in INR only (Indian audience).

9. Rate limit management: One image per tool call. Fresh conversation per chapter
   pair avoids context accumulation tax. Never batch images in one turn.

10. Phase 2 result: Kannada OCR is clean — zero errors across 476 pages.
    Only contamination in KN file is the garbled parenthesized English terms
    (Phase 1), not the body text.
